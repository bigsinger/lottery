# 抽奖网页程序 - 需求文档

## 1. 项目概述

开发一个基于 HTML/CSS/JS 的转盘抽奖网页程序，支持个性化配置、本地存储、多模式切换、多设备同步，可部署在简单的虚拟主机上。

### 1.1 产品定位

本程序是一款灵活的转盘抽奖工具，支持两种主要应用场景：

**场景一：抽奖品**
- 适用于各类活动、促销、年会等场景
- 设置奖品列表进行随机抽奖

**场景二：抽人**
- 适用于公司、班级、团队等场景
- 设置人员名单进行随机抽取
- 典型应用：年会抽奖员工、班级点名、团队分配任务、会议选发言人

## 2. 技术约束

### 2.1 技术栈限制

- ✅ HTML5 + CSS3 + JavaScript（原生）
- ✅ PHP（可选，用于多设备同步）
- ❌ 不允许使用复杂框架
- ✅ 使用相对路径，确保本地和服务器效果一致

### 2.2 部署要求

- 纯静态文件，无需后端服务（基础功能）
- 上传 src 目录即可立即生效
- 本地双击 HTML 即可预览，所见即所得
- 支持 PHP 虚拟主机实现多设备同步

### 2.3 兼容性

- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 支持移动端响应式布局
- 已优化手机端转盘渲染问题

## 5. 数据结构

### 5.1 配置数据结构（本地存储）

```json
{
  "mode": "default",
  "title": "幸运抽奖",
  "prizes": [
    {
      "id": "abc123",
      "text": "一等奖",
      "icon": "🏆",
      "color": "#e74c3c",
      "weight": 10,
      "maxWins": 1,
      "winCount": 0
    }
  ],
  "updatedAt": "2026-04-09T00:00:00.000Z"
}
```

### 5.2 服务端配置数据（多设备同步）

存储路径：`user/{username}.json`

一个 user 的 JSON 文件包含多个 mode 配置：

```json
{
  "user": "张三",
  "modes": {
    "default": {
      "title": "幸运抽奖",
      "prizes": [...],
      "updatedAt": "..."
    },
    "team1": {
      "title": "团队抽奖",
      "prizes": [...],
      "updatedAt": "..."
    },
    "class1": {
      "title": "班级抽奖",
      "prizes": [...],
      "updatedAt": "..."
    }
  },
  "updatedAt": "2026-04-09T00:00:00.000Z"
}
```

**参数说明**：
- `user` 参数：指定 JSON 配置文件名（如 `user/张三.json`）
- `mode` 参数：指定在该 JSON 文件中使用哪个 mode 配置
- 如果指定的 mode 不存在，保存时会自动创建

### 5.3 数据字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 选项唯一标识 |
| text | string | 选项显示名称 |
| icon | string | 预制图标（Emoji） |
| color | string | 扇形背景颜色 |
| weight | number | 权重（0-100百分比） |
| maxWins | number | 最大中奖次数 |
| winCount | number | 当前中奖次数 |

### 5.4 本地存储 Key

```
lottery_config_{mode}  // 配置数据
lottery_history_{mode}  // 抽奖历史
```

## 7. 非功能需求

### 7.1 性能要求

- 首次加载时间 < 2秒
- 转盘动画流畅（60fps）
- 本地存储读写即时响应
- 手机端渲染正常

### 7.2 用户体验

- 操作简单直观
- 配置实时保存
- 错误提示友好
- 支持键盘操作
- 中奖次数可视化
- 编辑自动滚动定位

### 7.3 安全性

- 本地存储数据不包含敏感信息
- 配置数据可随时清除
- PHP API 有 .htaccess 保护
- 用户配置目录禁止直接访问
