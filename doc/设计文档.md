# 抽奖网页程序 - 设计文档

## 0. 产品定位

### 0.1 核心功能

本程序是一款灵活的转盘抽奖工具，支持两种主要应用场景：

**场景一：抽奖品**
- 适用于各类活动、促销、年会等场景
- 设置奖品列表进行随机抽奖
- 支持权重控制中奖概率

**场景二：抽人**
- 适用于公司、班级、团队等场景
- 设置人员名单进行随机抽取
- 典型应用：年会抽奖员工、班级随机点名、团队随机分配任务、会议随机选择发言人
- 支持批量导入人员名单（每行一个姓名）
- 支持设置最大中奖次数，避免重复抽中同一人
- 支持权重设置（权重0=不参与抽取，权重100=必定抽中）

### 0.2 功能特性

- 纯前端实现，无需后端服务（基础功能）
- PHP 后端支持多设备配置同步
- 支持多模式独立配置
- 支持权重概率控制（0-100百分比）
- 支持批量导入名单
- 支持中奖次数限制
- 支持中奖次数可视化显示
- 响应式设计，支持移动端（已优化手机端渲染）
- 文字在上图标在下，显示区域更大

## 3. 核心模块设计

### 3.1 存储管理器 (StorageManager)

**职责**：统一管理配置和历史的存储，支持本地和服务端两种模式

**核心方法**：
```javascript
class StorageManager {
  // 判断存储模式
  isServerMode()               // 检查是否有 user 参数
  
  // 配置管理（本地）
  saveConfig(mode, config)     // 保存配置到 localStorage
  loadConfig(mode)             // 从 localStorage 加载配置
  clearConfig(mode)            // 清除配置
  
  // 配置管理（服务端）
  saveToServer(user, config)   // 通过 PHP API 保存配置
  loadFromServer(user)         // 通过 PHP API 加载配置
  
  // 历史管理
  saveHistory(mode, history)   // 保存历史
  loadHistory(mode)            // 加载历史
  clearHistory(mode)           // 清除历史
  
  // 当前模式
  setCurrentMode(mode)         // 设置当前模式
  getCurrentMode()             // 获取当前模式
  getCurrentUser()             // 获取当前用户（多设备同步）
}
```

```
    ├─ 初始化 StorageManager
    │   ├─ 有 user 参数 → 服务端模式
    │   └─ 无 user 参数 → 本地模式
    │
    ├─ 加载配置
    │   ├─ 服务端模式 → PHP API 加载
    │   └─ 本地模式 → localStorage 加载
    │
    ├─ 初始化 Turntable
    │   └─ 绘制转盘（文字在上图标在下）
    │
    └─ 初始化 App
```

### 4.2 抽奖流程

```
用户点击「开始抽奖」
    │
    ├─ 检查是否正在旋转
    │   └─ 是 → 忽略
    │
    ├─ 检查是否所有奖品已抽完
    │   └─ 是 → 提示"全部抽完，请重置抽奖结果重新开始"
    │
    ├─ 根据权重计算目标位置
    │   └─ 权重越大，概率越高

    ├─ 调用 Turntable.spin(targetIndex)
    │   └─ 执行旋转动画（CSS Transform）
    │
    ├─ 动画结束
    │   ├─ 显示中奖结果
    │   ├─ 更新中奖次数
    │   └─ 保存到历史记录
    │
    └─ 更新存储
    │   ├─ 服务端模式 → PHP API
    │   └─ 本地模式 → localStorage
```

### 4.3 配置保存流程

```
用户修改配置
    │
    ├─ 验证配置合法性
    │   ├─ 奖品数量 ≥ 2
    │   ├─ 权重范围 0-100
    │   └─ 最大中奖次数 ≥ 0

    ├─ 更新 ConfigManager
    │
```

## 6. 权重算法设计

### 6.1 权重转概率

```javascript
// 权重范围：0-100（百分比）
// 权重为 0 表示永不中奖
// 权重为 100 表示必中

function calculateProbability(prizes) {
  // 过滤掉权重为 0 的奖品
  const validPrizes = prizes.filter(p => p.weight > 0);
  
  // 计算总权重
  const totalWeight = validPrizes.reduce((sum, p) => sum + p.weight, 0);
  
  // 计算每个奖品的概率
  return validPrizes.map(p => ({
    ...p,
    probability: p.weight / totalWeight
  }));
}
```

### 6.2 随机选择算法

```javascript
function selectPrize(prizes) {
  // 过滤已达到最大中奖次数的奖品
  const availablePrizes = prizes.filter(
    p => p.winCount < p.maxWins
  );
  
  // 检查是否有可用奖品
  if (availablePrizes.length === 0) {
    throw new Error('全部抽完，请重置抽奖结果重新开始');
  }
  
  // 计算概率
  const withProb = calculateProbability(availablePrizes);
  
  // 随机选择
  const random = Math.random();
  let cumulative = 0;
  
  for (const prize of withProb) {
    cumulative += prize.probability;
    if (random <= cumulative) {
      return prize;
    }
  }
  
  return withProb[withProb.length - 1];
}
```

## 8. 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 奖品数量不足 | 提示"请至少添加2个奖品" |
| 权重全为0 | 提示"请设置至少一个权重大于0的奖品" |
| 全部抽完 | 提示"全部抽完，请重置抽奖结果重新开始" |
| localStorage 不可用 | 使用内存存储，提示"本地存储不可用" |
| Canvas 不支持 | 提示"您的浏览器不支持 Canvas" |
| PHP API 失败 | 回退到本地存储，提示"服务端保存失败" |

## 9. 性能优化

### 9.1 渲染优化

- Canvas 绑定 DPI 缩放，避免模糊
- 使用 CSS Transform 进行动画（硬件加速）
- 窗口 resize 时防抖处理
- 手机端特殊尺寸计算，确保渲染完整

### 9.2 存储优化

- 配置变更时防抖保存
- 历史记录限制最大条数（100条）
- 服务端存储使用 JSON 文件，无需数据库

### 9.3 文字渲染优化

- 文字在上、图标在下，文字显示区域更大
- 动态字体大小：小屏9px、中屏10px、大屏12-14px
